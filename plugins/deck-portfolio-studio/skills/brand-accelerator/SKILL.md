# Personal Brand Accelerator — Design Skill

마케터의 **개인 포트폴리오 비주얼 아이덴티티**로 채용·자기 PR·콘텐츠 자산을 제작하는 스킬입니다.
검증된 디자인 시스템(컬러 토큰·타이포·컴포넌트)과 그로스 스토리 서사 구조를 제공하고, **개인 데이터는 사용자가 채워 넣는 템플릿** 방식입니다.

> 이 스킬은 팀 공유용 템플릿입니다. 특정 개인의 실명·연락처·매출 수치는 들어있지 않으며,
> 아래 `## 프로필 / 데이터` 섹션을 **본인 정보로 채운 뒤** 사용하세요.

## 시작하기

1. 처음 호출되면 사용자에게 **본인 프로필/성과 데이터**를 한 번에 받습니다(`## 프로필 / 데이터` 양식 참고). 받은 값은 이후 자산 전반에 일관되게 사용합니다.
2. 컬러는 아래 `## 비주얼 토큰`의 CSS 변수를 사용하세요. 직접 hex를 박지 말고 토큰 이름을 쓰세요(`var(--blue-500)`, `var(--ink-800)` 등).
3. 슬라이드 덱은 `## 슬라이드 구조`의 4단 서사를 따라 구성합니다. 1920×1080 또는 1280×720, deck-stage 컴포넌트 사용.
4. 본인 작업물 이미지·로고가 있으면 프로젝트 폴더의 `assets/`에서 복사해 사용합니다(없으면 플레이스홀더 박스).

## 사용자가 가이드 없이 호출했을 때

먼저 무엇을 만들지 묻습니다:

- **무엇을 만드시나요?** (슬라이드 덱 · 단일 페이지 포트폴리오 · 인스타 캐러셀 · 명함 · 이력서 PDF · 컨택용 미니덱)
- **용도는?** (특정 회사 면접용 · 일반 자기소개 · 외부 강연 · 컨택용)
- **분량은?** (몇 슬라이드 / 몇 페이지)
- **강조할 성과는?** (전체 그로스 서사 · 특정 단계 · 특정 작업물)
- **프로필/성과 데이터** — 아직 안 받았으면 `## 프로필 / 데이터` 양식으로 한 번에 받기

답변을 받으면 디자인 전문가로서 다음을 검토:
- 메인 카피 톤 (1인칭 + 결과부터 + 데이터로 받침)
- 컬러 토큰 일관 사용 (blue-500 강조, mint-500 성장, rose-500 손실)
- 헤드라인 강조의 단일 사용 (`u-blue` 클래스를 한 헤드라인에 한 단어만)
- 이모지 절제 (섹션 헤더·단계 인디케이터만, 본문엔 X)
- 카드 패턴 재사용 (`.card`, `.metric` 등 시스템 클래스)

## 출력 형식

- **슬라이드/시각 자료/프로토타입:** 정적 HTML 파일 생성 → 사용자가 바로 열어볼 수 있게 제시
- **프로덕션 코드(실제 웹사이트/앱):** 토큰을 직접 박지 말고 사용자 시스템 변수로 옮겨 적용

## 프로필 / 데이터 (사용자가 채우는 양식)

> 아래 `<...>` 자리를 본인 정보로 교체해서 쓰세요. **추측으로 채우지 말고**, 값이 없으면 사용자에게 물어보세요.

- 이름: `<이름>` (영문 `<English Name>`), `<출생연도>`, `<거주지>`
- 경력: `<N년 M개월>` (`<직무: 그로스/퍼포먼스 마케터 등>`)
- 연락처: 이메일 `<email>` · 전화 `<phone>` · `<기타 채널>`
- 현재 직장: `<회사 · 직함 (기간)>`
- 학력: `<학교 · 전공>`
- 어학/자격: `<TOEIC 등 · 자격증>`

**핵심 헤드라인 성과** (정확한 수치만, 없으면 비워두고 사용자 확인):
- `<지표 1: 예) 앱 월매출 성장 N배 (A → B, 기간)>`
- `<지표 2: 예) CRM 추가매출 / 휴면 재활성화>`
- `<지표 3: 예) 월 광고 집행 규모 · 채널 수>`
- `<지표 4: 예) 팀 빌딩 / 전환>`

**그로스 스토리 단계** (병목→발견→실행→결과의 연속, 단계마다 이전→이후 수치):
1. `🎯 <1단계 주제> (<핵심 지표 변화>)`
2. `🏅 <2단계 주제> (<핵심 지표 변화>)`
3. `⚖️ <3단계 주제> (<핵심 지표 변화>)`
4. `🎲 <4단계 주제> (<핵심 지표 변화>)`
5. `🤝 <5단계 주제> (<핵심 지표 변화>)`
6. `🗺️ <6단계 주제> (<핵심 지표 변화>)`

**스킬 자기평가:** 상급 `<...>` / 중급 `<...>` / 하급 `<...>`

> ⚠️ 개인정보(연락처)와 회사 내부 매출 수치는 **공개 repo·웹사이트에 그대로 올리지 마세요.**
> 외부 공개용 자산이면 민감 수치는 지수·성장률(%)·상대값으로 바꾸거나 비공개 처리하세요.

## 비주얼 토큰

CSS 변수로 선언해 `var(...)`로 참조합니다(직접 hex 금지).

```css
:root {
  /* 코발트 블루 (강조·CTA·핵심 헤드라인) */
  --blue-700:#1b3b8f; --blue-500:#2f5bff; --blue-300:#8fa8ff;
  /* 잉크 블랙 (본문·헤딩) */
  --ink-900:#0c0f1a; --ink-800:#1a1f2e; --ink-600:#3a4254; --ink-400:#6b7385;
  /* 그로스 (성장·긍정 지표) */
  --mint-500:#16c79a;
  /* 손실·하락 지표 */
  --rose-500:#ff5470;
  /* 표면 */
  --bg:#ffffff; --surface:#f5f7fb; --line:#e6e9f2;
  --radius:16px;
}
```

폰트는 **Pretendard** 고정(웹폰트 CDN 또는 시스템 폴백). 한글 가독성 우선.

## 슬라이드 구조 (4단 서사)

개인 포트폴리오 덱·페이지의 기본 골격:

```
1. 헤드라인 (결과 임팩트 한 줄)
2. 병목 / 현황 — 무엇이 문제였는지 2~4문장, 발견을 인용구(>)로
3. 실행 — 무엇을 했는지 담담하게 1~3문장 + 곧바로 데이터 표/지표
4. 결과 + 인사이트 — 표 위 캡션 / 표 / 표 아래 callout(> ℹ️ / > ✅ / > 💡) / 다음 단계로 잇는 한 문장
```

- 수치는 **이전 → 이후 + Δ%** 패턴(`7,988원 → 2,169원, -72.8%`).
- 단계 전환은 `> 💡 N단계 → N+1단계 ...` callout으로 연결.
- 표는 **비교 표가 디폴트**(외주 vs 직접, 입사 전 vs 후, 컨트롤 vs 베리언트).

## 디자인 원칙 (한 줄 요약)

> **"숫자로 말하는 자신감, 결과로 증명하는 겸손함."**
>
> 코발트 블루 + 잉크 블랙. 모든 주장은 정량 데이터로 받칩니다. 헤드라인은 결과로, 본문은 과정으로, 끝은 학습으로.
> 이모지는 섹션 헤더와 단계 인디케이터에만. 그림자는 거의 안 보이게, 코너는 16px, 폰트는 Pretendard.

## 컴포넌트 카탈로그 (시스템 클래스)

- `.card` — 기본 카드(흰 배경, `--line` 테두리, `--radius`)
- `.metric` — 큰 수치 + 라벨 + Δ% (성장은 `--mint-500`, 하락은 `--rose-500`)
- `table` — 비교 표(헤더 `--surface`, 강조 셀 `--blue-500`)
- `.quote` — 인용구(`>` 스타일, 발견·고객 목소리)
- `.flow` — 단계 인디케이터(그로스 스토리 1→6)
- `u-blue` / `u-blue-mark` — 헤드라인 한 단어 강조(슬라이드당 1회)
